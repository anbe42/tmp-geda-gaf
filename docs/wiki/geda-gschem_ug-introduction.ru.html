Введение

gschem — это программа для схемотехнического проектирования, входящая в набор утилит gEDA (GPL Electronic Design Automation). Она используется для создания принципиальных схем электронных устройств, описывающих логическую структуру их электрических цепей. Схемы составляются из символов, являющихся представлением различных компонентов устройства. Они либо берутся из стандартной библиотеки, либо создаются самим пользователем. Электрические связи между компонентами представляются соединениями (проводниками). Схему можно вывести в PostScript-файл для печати или преобразования в другой выходной формат. Поддерживается и вывод изображений в нескольких форматах.

Помимо создания схем, gschem может использоваться для редактирования символов, используемых в схемах, а также для черчения структурных и функциональных схем для проектов электронных устройств.

Общие замечания

В этом руководстве для описания объектов, которые можно увидеть в программе gschem, и действий, которые можно в ней предпринять, используется несколько соглашений по оформлению.

  • При введении новых ключевых терминов и понятий используется курсив. Например: “gschem принимает несколько аргументов командной строки.”
  • Названия клавиш клавиатуры и определения их сочетаний выделяются жирным шрифтом и берутся в угловые скобки (”<” и ”>”). Для указания одновременного нажатия двух или более клавиш используется знак «+». Например: “Для выхода из gschem нажмите <Alt+Q>.”
  • Пункты различных меню gschem обозначаются жирным шрифтом, а для описания порядка выбора пунктов меню используется символ “→”. Например: “Для отображения списка привязок клавиш выберите Help→Hotkeys.”

Получение дополнительной помощи

Если у вас есть какой-то вопрос по использованию gschem, можете задать его в одном из списков рассылки gEDA. Кроме того, вопрос можно задать на странице geda в Launchpad, и кто-нибудь из разработчиков попытается на него ответить.

Если вы думаете, что нашли ошибку в gschem, заполните, пожалуйста, отчёт об ошибке в Системе отслеживания ошибок Launchpad и добавьте для него тег “gschem”. Если вы сможете подробно написать, как воспроизвести найденную ошибку, разработчикам будет легче быстро решить вашу проблему.
